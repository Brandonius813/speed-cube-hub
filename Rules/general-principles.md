# General Principles

- **Simplicity over cleverness.** The simplest solution that works is always the right solution.
- **Working beats perfect.** A feature that works simply is better than a feature that's elegant but fragile.
- **Ask, don't assume.** When in doubt about what the user wants, ask a clear question rather than making a choice for them.
- **Respect what works.** Never touch code that is already working unless it is directly in the way of the current task.
- **Be honest about uncertainty.** Saying "I'm not sure" is always better than confidently doing the wrong thing.
