# Make the Smallest Change Possible

Always find the solution that touches the least amount of existing code.

- Do not refactor working code while adding a new feature
- Do not rename things that already work
- Do not reorganize files unless specifically asked
- Do not upgrade packages unless specifically asked
- Do not add new dependencies without explaining why

If there are two ways to solve a problem, always choose the one that changes fewer files — even if the other approach is more elegant or "correct."

Ask yourself before every change: *"Is there a way to do this that touches less code?"*
