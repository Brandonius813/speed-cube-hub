# Explain Everything in Plain English

When you do anything, explain:
- **What** you are doing
- **Why** you are doing it this way
- **What file(s)** you are changing
- **What it will look like** when it works

Avoid jargon. If you must use a technical term, define it immediately.

**Good:**
> "I'm creating a new page for the login form. This means adding one new file at `app/login/page.tsx`. It won't change anything that already exists."

**Bad:**
> "I'll scaffold the auth route and wire up the session middleware."
